# 选题与流水线校验简化需求

日期：2026-07-22

## 修改目标

简化定时任务和手动任务的选题、配图、排版及草稿校验逻辑，删除重复且容易阻塞生产的检查。

目标是支持同一账号每天生成多篇草稿。只保留必要的业务约束、路径安全、账号配置、微信 API 错误处理和轻量草稿防重。

## 一、选题只保留 3 项校验

### 1. 类别约束

- 选题必须属于账号档案 `topic_discovery.categories` 中允许的类别。
- 不检查受影响人群及其他选题元数据字段。

### 2. 时效约束

- 只取 48 小时内的新闻热点。
- 删除原有 `window_hours` 和 `fallback_hours` 双层窗口，账号档案统一使用 `max_age_hours: 48`。
- 自动选题只保存热点本身的一个 `published_at`，不再要求每个证据 URL 都提供发布时间。
- `published_at` 只在确定选题、写入历史前校验一次。
- 选题一旦通过，后续 `prepare`、`finish` 和草稿阶段不再重复检查时效，避免制作过程中因时间流逝被阻塞。

### 3. 七天语义去重

- 自动选出热点后，将以下内容写入账号级选题历史：
  - `topic`：最终选题或标题。
  - `event_focus`：由 Agent 写的一句简短事件重点，说明“哪个主体发生了什么核心变化”。
  - `selected_at`：选题时间。
- 只读取最近 7 天（168 小时）的历史。
- 删除原有 30 天去重窗口。
- 删除字符 bigram、Jaccard、overlap 等机械相似度算法。
- 去重不要求标题或 `event_focus` 文本完全相同。
- Agent 在选题前读取近 7 天的 `event_focus`，按语义判断是否为同一核心事件；同一事件只是改标题、换说法或换写作角度，仍应拒绝。
- 不做机械相似度阈值拦截。若无法确定是否重复，默认允许继续，避免误杀新选题。

## 二、删除来源文件和相关校验

- 完全删除 `sources.md` 产物，不再生成、记录或校验。
- 删除 `fact-check` 阶段中对 `sources.md` 的依赖；如 `fact-check` 仅服务于该文件，可直接删除该阶段。
- 不要求至少两个不同来源域名。
- 不要求证据 URL 与正文或 discover 阶段逐项对应。
- 不要求每条证据包含带时区发布时间。
- 不要求受影响人群字段。
- Agent 写作时仍应依据真实联网资料，不得编造事实，但不再通过 `sources.md` 做机械门禁。

## 三、正文配图改为可降级的 0—3 张

- 正文配图目标为至少 1 张，但不是硬门禁。
- 根据文章内容和可配图信息灵活生成，允许 0—3 张，绝对不能超过 3 张。
- 信息不适合配图、没有必要配图，或生图失败时，可以 0 张继续。
- 单张生图失败后只重试一次；重试仍失败则跳过该图，不阻塞后续流程。
- 全部配图失败时，将 `illustrations` 标记为 `skipped` 或等价的可继续状态，不得标记为失败。
- 删除 Agent 手工填写的 `image_count` 门禁。
- 删除 `article.md` 图片引用数量与 `image_count` 一致性检查。
- 运行器只统计 `article.md` 中实际存在的图片引用：
  - 0—3 张允许继续。
  - 超过 3 张拒绝。
  - 引用路径越出当前工作区时拒绝。
  - 引用文件不存在时删除该图片引用并继续；不得阻塞整篇草稿。

### PNG、JPG 等格式处理

- 支持 PNG、JPEG/JPG、GIF、WebP，不限定生成结果必须是某一种格式。
- 不再用文件扩展名或 HTTP 声明类型与真实格式不一致作为硬性失败条件。
- 上传前根据文件实际字节识别图片类型，并据此设置 MIME 类型。
- 若文件扩展名与实际类型不一致，自动使用正确扩展名生成上传文件名，例如实际为 JPEG 的 `.png` 文件按 `.jpg` 上传，不修改文章内容语义。
- 删除“扩展名、magic bytes、Pillow 解码结果三者必须一致”的门禁。
- 无法识别或微信 API 不接受的正文图片，删除该图片引用后继续上传其余正文；没有可用正文图时按无图文章继续。
- 图片大小不作为整篇文章的硬门禁。单张图片超过微信接口可接受范围时，跳过该图片并继续，不得阻塞整篇草稿。
- 上述降级只针对正文配图；封面仍按现有“生成封面或账号默认封面”逻辑处理。

## 四、删除 validate、预览和文件哈希

- 删除整个 `validate` 阶段。
- 删除严格 HTML validator 调用，不再要求零 ERROR、零 WARNING。
- 删除预览生成及 `article_preview.html` 产物。
- 删除 `leaf_count`。
- 删除 `html_sha256`、`preview_sha256`、`article_sha256`、图片哈希和草稿输入全文哈希。
- 删除基于文件哈希的 format、cover、validate、draft checkpoint 复用和拦截逻辑。
- 排版后只做轻量检查：
  - `article.html` 文件存在且非空。
  - 不包含明确的未替换占位符。
  - 正文图片路径不得越界。
- 不再因样式 warning、节点数量、文件变化或缓存指纹不一致阻塞草稿。

## 五、同账号每天多篇与轻量草稿防重

- 删除“同账号今天已有 `state=drafted` 就静默退出”的规则。
- 同一账号一天内允许创建多篇草稿，定时任务和手动任务均不得按日期拦截。
- 每次 `init` 创建新的随机 `run_id`，并写入 `job.json`。
- 草稿结果只记录：`run_id`、账号、草稿 `media_id`、创建结果和时间。
- 同一个 `run_id` 已有成功且包含非空 `draft_media_id` 的结果时，直接返回原结果，不再次调用 `draft/add`。
- 新的 `run_id` 即视为新的文章任务，即使与当天已有草稿属于同一账号，也允许继续创建草稿。
- 不使用正文、来源、HTML、图片或封面的文件哈希判断是否重复。
- 微信 `draft/add` 请求遇到 timeout、EOF、连接重置或响应不完整时，仍标记为 `uncertain`，不得自动重发；该规则只防止同一次未知结果重复创建，不妨碍新任务。
- 请求前即可确定的错误，例如账号配置或环境变量缺失，可修复后在同一 `run_id` 下重试。

## 六、保留的必要检查

- 选题类别属于账号允许类别。
- 自动热点在选题时属于 48 小时内。
- Agent 基于 `event_focus` 做近 7 天语义去重。
- 正文标题、正文基本存在性及 1500—4000 字门禁保持不变。
- 正文配图最多 3 张。
- 本地文件和图片路径不能越出当前工作区。
- 封面生成或账号默认封面兜底逻辑。
- 账号配置完整性检查。
- AppID、AppSecret 等环境变量检查。
- access token 获取。
- 微信 API 错误处理。
- 微信返回结果必须包含必要字段。
- 草稿请求结果不确定时标记为 `uncertain`，不自动重发。
- 同一 `run_id` 的成功草稿不得重复发送。

## 七、删除的原有校验汇总

- 至少 2 个不同来源域名。
- 每条证据必须有带时区的发布时间。
- 不能是未来时间的单独门禁；只判断热点是否属于最近 48 小时。
- 首选窗口与回退窗口。
- 必须明确受影响人群。
- 近 30 天字符 bigram 相似度去重。
- `sources.md` 及全部相关校验。
- `image_count` 与 Markdown/HTML 图片数量一致性。
- 图片扩展名、声明类型、magic bytes、Pillow 结果必须一致。
- 整个 validate 阶段。
- HTML 严格 ERROR/WARNING 门禁。
- 预览生成。
- `leaf_count`。
- 文件哈希和基于文件哈希的 checkpoint。
- 当天已有草稿后不再运行的日期级拦截。

## 九、测试与验收

至少覆盖以下场景：

1. 自动热点类别正确、发布时间在 48 小时内，可以写入选题。
2. 热点超过 48 小时被拒绝。
3. 不提供两个来源、不提供受影响人群，也可以写入选题。
4. 近 7 天历史会提供给 Agent；语义相同事件被 Agent 拒绝，无法确定时允许继续。
5. `sources.md` 不存在时，`prepare` 和 `finish` 正常运行。
6. 正文 0 张图片时可以创建草稿。
7. 正文 1—3 张图片时可以创建草稿。
8. 生图失败两次后降级为无图，不阻塞草稿。
9. 实际 JPEG 但扩展名为 `.png` 的图片会按 JPEG 上传。
10. 单张图片损坏或过大时跳过该图，其余正文继续。
11. 不生成预览，不运行 validate，不记录文件哈希。
12. 同一账号同一天使用两个不同 `run_id`，可以连续创建两篇草稿。
13. 同一个 `run_id` 已成功时再次执行，只返回原 `draft_media_id`，不调用第二次 `draft/add`。
14. 同一个 `run_id` 的草稿结果为 `uncertain` 时停止，不自动重发。
15. 正文图片路径越界时仍然拒绝。
16. 账号配置或环境变量缺失时仍然在请求前失败。
