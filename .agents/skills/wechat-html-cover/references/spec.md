# 封面 JSON 规格

封面提供三套并列模板，不在 Skill 内绑定账号，也不跟随正文主题换色：

- `signal-editorial`：暖白编辑海报，黑色衬线标题配钴蓝竖轨和黄色信号点。
- `night-signal`：深蓝科技海报，暖白粗体标题配橙色竖标和青色细线。
- `redaction-poster`：暖白信息海报，黑色标题块配朱红主题词和底边。

```json
{
  "template": "redaction-poster",
  "theme": "olive-journal",
  "eyebrow": "AI 与产业观察",
  "title": "模型变便宜之后，真正昂贵的是什么",
  "title_lines": ["模型变便宜之后，", "真正昂贵的是什么"],
  "highlights": ["模型"],
  "subtitle": "企业竞争正在从参数转向工作流程"
}
```

流水线不得手工填写这份 JSON。使用 `scripts/build_cover_spec.py --article ...` 生成；下面字段用于解释校验契约和维护模板。

- `template`：三套注册模板之一；具体账号映射以后再配置。
- `theme`：当前文章已固定的六套主题标识之一，仅用于任务追踪，封面使用模板自身固定配色。
- `eyebrow`：栏目或文章类型，最多 18 字。
- `title`：最终文章标题，最多 32 字。
- `title_lines`：两行或三行，拼接后必须与 `title` 完全一致；每行最多 18 字。超过视觉密度门槛时自动使用三行。
- `highlights`：0—1 个标题中的原文短语，最多 8 字。只有 `redaction-poster` 使用该词高亮主题；其余模板必须为空。
- `subtitle`：正文已有的一句有限判断，最多 40 字。

禁止加入日期、期号、模型版本、随机字母、伪代码和正文不存在的数据。标题分行只改变视觉结构，不得改变文字。

生成器按字符视觉宽度平衡分行，优先在中文标点后断开且不拆开英文单词。它再根据最宽一行和模板安全区计算字号；32 字压力标题自动使用三行。`redaction-poster` 最多高亮第一行一个主题词，另外两套模板不高亮。

三套模板的标题、主题词与各自背景均必须达到至少 4.5:1 的确定性对比度门槛。该检查只计算颜色，不调用视觉模型。
