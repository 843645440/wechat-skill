# Snap Chromium 暂存与封面内容验证

## 适用场景

Chrome/Chromium 能启动并生成 1410×600 PNG，但目标工作区位于隐藏目录（例如 `~/.hermes/...`），或生成结果疑似浏览器错误页。

## 最小复现顺序

1. 保留浏览器启动器原路径，例如 `/snap/bin/chromium`；不要 `resolve()`。
2. 在 `$HOME` 下创建普通非隐藏暂存目录，例如 `$HOME/wechat-cover-tmp/render-XXXX/`。
3. 将以下三类路径全部放在暂存目录：
   - 输入 `cover.html`
   - `--user-data-dir` 浏览器 profile
   - `--screenshot` 输出 `cover.png`
4. 使用原浏览器、原 HTML 做一次真实截图，不先更换浏览器，也不引入 HTTP 服务。
5. 验证通过后，再把 PNG 原子移动到隐藏工作区；无论成功失败都清理本轮暂存子目录。

## 必须验证的三层结果

1. **文件层**：PNG 存在、签名有效、尺寸严格为 1410×600。
2. **内容层**：读取实际像素或做视觉检查，排除：
   - `ERR_ACCESS_DENIED`
   - `ERR_FILE_NOT_FOUND`
   - 空白页、浏览器默认错误页
   - 标题缺失、乱码、裁切、溢出
3. **回归层**：覆盖“浏览器退出前写入”和“退出瞬间才写入”两种情况；跑完整封面及流水线测试。

## 失败处理

- 内容未核验：不得上传，也不得报告成功。
- 浏览器技术失败：按上层流水线限制最多重试一次。
- 仍失败：使用账号已配置的默认 `thumb_media_id`；没有兜底素材则停止草稿创建。

## 不要做

- 不要因为沙箱路径问题立刻切换多个浏览器实现。
- 不要先搭 HTTP 服务；本地非隐藏目录能解决时，它只会增加端口、线程和清理故障面。
- 不要用文件大小或尺寸代替内容验证；错误页同样可能是合法的 1410×600 PNG。
