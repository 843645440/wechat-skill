# 中文标题封面可读性门禁

## 适用场景

当封面 PNG、尺寸、色彩对比度都校验通过，但实际画面中的中文标题出现笔画膨胀、字符粘连、负字距过紧或缩略图不可读时，使用本门禁。技术渲染成功不等于视觉可用。

## 常见高风险组合

Linux/Chromium 使用中文后备字体或合成粗体时，以下组合容易让汉字看起来互相覆盖：

- 标题字号约 100px 以上；
- `font-weight: 900`；
- 负字距（例如 `letter-spacing: -0.045em`）；
- 两行标题中单行包含 8—10 个全宽汉字。

这个问题可能同时出现在多套模板中。若换模板后标题仍以相同方式粘连，不要继续盲目换色或换构图，应把问题归到标题排印参数。

## 封面单测流程

1. 保持文章标题逐字不变，不用删字、改词或隐藏字符规避。
2. 先渲染注册模板并查看实际像素；`visual_check: not-required`、PNG 签名、1410×600 尺寸和对比度通过，都不能替代可读性检查。
3. 若只做封面效果测试，不重跑正文、不创建或更新草稿。候选图放到独立路径，避免覆盖已上传封面。
4. 若所有注册模板都因同一排印参数失败，可在**单张本地产物**中做受控收敛：
   - 标题字号优先试 80—90px；
   - 字重优先试 600—700；
   - 取消负字距，必要时使用 `0.06em`—`0.10em` 正字距；
   - 仅微调标题块位置以平衡上下留白；
   - 不改标题文本、栏目眉题和事实性副文案。
5. 使用与正式渲染相同的 Chromium，以 1410×600 截图。
6. 最终逐字核对标题，并检查：字符是否粘连、行间是否碰撞、是否裁切、是否乱码、缩略图是否还能快速读完。
7. 另存为 `cover-optimized.png` 一类明确名称。只有用户明确要求替换草稿时，才进入重新上传或草稿更新流程。

## 通过标准

- 标题与文章标题逐字一致；
- 无字符重叠、明显粘连、裁切或乱码；
- 重点色不降低文字辨识度；
- 1410×600，且标题位于单一高对比安全区；
- 在缩略图尺度仍能辨认标题两行的主要信息；
- 若仅做封面单测，报告中明确说明草稿未更新。

## 长期模板维护建议

一次性产物调整只能用于封面效果验证。若同一问题在多篇文章或多套模板中复现，应在模板维护任务中增加代表性中文压力标题，使用实际浏览器字体测量结果计算字号、字重和字距，并补充像素回归样例；不要把一次性 HTML 覆盖当成永久模板修复。
