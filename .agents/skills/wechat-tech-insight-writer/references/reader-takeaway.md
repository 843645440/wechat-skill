# 读者带走什么（反「白看一场」）

强情感和观点不等于有用。读完若只能记住「作者很烦」，算失败。  
每篇必须让目标读者**能拿走至少一样可执行或可判断的东西**。

## 硬门禁：信息增量

正文除情绪与站队外，至少包含下列中的 **3 类**（多类更好）：

| 类型 | 要求 | 反例 |
|------|------|------|
| **可核验事实锚** | 时间/主体/动作/来源性质（公开报道、公告） | 只有态度没有事 |
| **机制一层** | 为什么会这样（流程、激励、约束） | 只骂通稿 |
| **错误对照** | 常见误读 vs 更准的读法（1—3 组） | 只有正确句没有对照 |
| **可执行动作** | 读者今晚/本周能做的具体事（清单 3—5 条，或决策树） | 「值得关注」 |
| **判断标准** | 怎样算过/怎样算还早（可检查） | 空泛「仍需观察」 |
| **边界** | 本文不谈什么、证据停在哪 | 假装全知 |

**「可执行动作」或「判断标准」至少占 1 类**——否则容易变成纯情绪散文。

## 推荐落地形态（嵌在文中，勿堆附录）

任选 1—2 种写进正文，不要机械凑满：

1. **验收/自检清单**（3—5 条短句，可照着勾）  
2. **会怎么开**（采购会/复盘会该问的 3 个问题）  
3. **别信这句 / 改信这句**（通稿金句 vs 你的改写）  
4. **分层读者动作**（负责人 / 一线 / 个人学习者各 1 条）  
5. **时间盒**（本周只做 X，先别做 Y）

## 与声口的关系

- 清单可以锋利，但必须**可执行**。  
- 禁止为了有用而编造数据、内部案例。  
- 禁止写成无态度的纯手册——仍要有「我」和站队。  
- 有用部分服务于 tension，不是另开百科。

## 自检（交付前）

- [ ] 遮住所有情绪句，是否还剩可带走的事实/清单/标准？  
- [ ] 读者若只做一件事，文中是否写清了「做哪件」？  
- [ ] 是否避免了「通稿复读 + 发牢骚」两端空？  

任一否 → 同轮补一节「你今晚可以做的」或「怎样算过」，再交付。
