# 内容表达能力契约

文章只输出语义 Markdown，绝不写排版 HTML。排版器会按本轮主题把这些结构转换为完整组件，因此写作时必须在“纯文字段落”和更合适的结构之间做一次选择。

## 选择顺序

| 内容关系 | 使用结构 | 使用条件 |
|---|---|---|
| 三行以上、同一口径的数据或对象比较 | Markdown 表格 | 至少 2 列且每个单元格都有可靠来源；缺项较多时不用 |
| 两个对象或阶段的同维度差异 | `comparison` 信息块候选 | 两侧各有 2—4 个可比点；正文先准确写明差异 |
| 2—4 个决定判断的数字 | `metrics` 信息块候选 | 数值已核实并原样进入正文 |
| 3—5 个连续环节 | `process` 信息块候选 | 存在真实先后或依赖关系，不能把并列观点伪装成流程 |
| 2—4 类受影响群体或关键判断 | `insight` 信息块候选 | 各项边界清楚且能在正文找到证据 |
| 并列事实、条件或限制 | Markdown 列表 | 项目属于同一层级，通常 3—6 项 |
| 核心判断或真实引用 | Markdown 引用 `>` | 判断已经在正文得到支持；不得虚构人物引用 |
| 普通论述 | 段落 | 内容没有稳定结构时保持文字，不为设计感硬拆 |

## 写作标记

- 在重要段落中用 `**短语**` 标出 1—2 个最值得扫读的事实、对象或判断；排版器会转换为当前主题的下划线或强调样式。
- 使用 `###` 表示章节内部确有必要的小标题。
- 表格使用标准 Markdown，列名必须具体，例如“企业 / 公开价格 / 适用场景”，不要使用“项目一 / 项目二”。
- 表格只承载可比较事实，解释、限制条件和判断仍放在表格前后正文中。
- 同一组信息只选择一种主要结构。已经用表格清楚表达的数据，不再生成重复的对比卡或指标卡。

## 事实边界

- 不得为了凑表格补造数值、单位、排名、时间或空缺项。
- 不同统计口径、日期或对象的数据不得放在同一列直接比较，除非正文明确解释差异。
- 来源只能支持定性描述时使用文字，不把模糊程度强行量化。
- 信息密度不足时允许全文没有表格和信息块；结构服务理解，不是装饰配额。
