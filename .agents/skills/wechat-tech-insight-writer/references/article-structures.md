# 文章结构池与轮换

只选一种主结构。所有结构必须围绕：

1. 一个核心问题  
2. 一句中心判断（我站哪边）  
3. 一个 tension  
4. 一个主导 `felt_sense`（主情绪）

**默认不是永远同一结构。** 流水线模式下必须先读  
`pipeline_job.py history --rotation`，再 `shape` 锁定，且遵守轮换硬规则（见 `wechat-content-pipeline/references/structure-rotation.md`）。

非流水线单篇：仍从池中选，并避免与你刚写过的上一篇同构。

## structure_id 池（写作必须落到其中一个）

| structure_id | 名称 | 逻辑顺序（强情感「我」贯穿） |
|--------------|------|------------------------------|
| `felt_essay` | 主观强情感随笔 | 情绪刺点 → 嵌入事实 → 我为什么在意 → 机制 → 镜头 → 站队 → 未解不适 |
| `conflict` | 冲突型 | 甩出矛盾 → 事实 → 两边压力 → 机制 → 我站哪边 → 未解 |
| `myth_bust` | 误区纠正 | 流行说法（我烦）→ 拆穿 → 真机制 → 谁靠误区获利 → 我的读法 |
| `workflow_day` | 一条工作流 | 旧痛点 → 工具插入点 → 压缩了什么 → 卡点 → 新累法 → 判断 |
| `judgment_first` | 判断前置 | 先站队金句 → 证据链 → 反方通稿 → 限制条件 → 收回判断 |
| `sting_list` | 清单刺点 | 总刺点 → 2–4 个真变化 → 1–2 个被吹大 → 我的取舍（禁止报告体小标题） |
| `qa_drive` | 自问自答 | 2–4 个真问题依次推进（自问自答，非虚构采访） |
| `quick_take` | 短快评 | 刺点 → 关键事实 → 一个机制 → 一个代价 → 收束（信息薄时用，勿注水） |
| `event_read` | 事件解读 | 刺点切入 → 发生了什么 → 为何此刻 → 拆解 → 利害镜头 → 有限判断 |
| `tech_explain` | 科技解释 | 场景 → 解决什么 → 原理够用即可 → 演示/产品/部署分层 → 成本责任 → 影响 |
| `company_compete` | 企业竞争 | 场景界定 → 策略 → 比较 → 约束 → 对人影响 → 变量 → 阶段性判断 |
| `industry_game` | 产业博弈 | 链条约束（技术/供应/标准/人才等）→ 去战争叙事 → 落地条件 |
| `tech_livelihood` | 科技民生 | 具体体验 → 原因 → 谁被压到 → 便利是否到达 → 边界 → 判断 |
| `non_invest_finance` | 非投资财经 | 经营问题 → 收支/供需机制 → 策略 → 影响 → 风险 → 经营判断（禁荐股） |

## 开头类型 opening_type

| 类型 | 写法 |
|------|------|
| `emotion_sting` | 我被刺到/发紧/兴奋的那一下 |
| `contrast` | 表面利好 vs 真实代价 |
| `myth` | 先甩流行说法再打 |
| `scene` | 具体流程现场（不编人物名） |
| `judgment_first` | 先抛站队判断 |
| `date_announce` | 日期+宣布（**不推荐**，轮换里尽量不用） |

## 结尾类型 ending_type

| 类型 | 写法 |
|------|------|
| `duty_point` | 落到签字/权限/验收责任点 |
| `unresolved` | 留下未消化的不适，不和解 |
| `actionable_question` | 一个可执行狠问题 |
| `hook_return` | 收回开头钩子 |
| `brief_approval` | 对真减负的短暂认可 + 条件 |

## 轮换硬规则（流水线与连载账号）

1. 近 **7** 篇：同一 `structure_id` **≤ 2** 次。  
2. 近 **3** 篇：尽量不重复同一 `structure_id`（有替代时禁止连用）。  
3. 近 **5** 篇：`opening_type` / `ending_type` **不重复**。  
4. 近 **5** 篇：同一 `tension_type` **≤ 2**。  
5. 近 **3** 篇：`felt_sense` 不雷同。  
6. `heading_count` ∈ {2,3,4,5}，禁止连续多篇相同。  
7. `body_band` short/mid/long 轮换。  

违反 = **交付失败**，同轮换结构重写；不要指望换排版主题蒙混。

## 快速选择（在轮换约束内）

- 默认在 **preferred_structures** 里挑最贴 tension 的，而不是永远 `felt_essay`。  
- 通稿假话重 → `myth_bust`  
- 利害撕裂 → `conflict`  
- 工具插入流程 → `workflow_day`  
- 信息薄 → `quick_take`  
- 需要清晰站队 → `judgment_first`  

## 失败结构（必须重写）

- 第三人称新闻扩写 + 克制展望  
- 与近文同一骨架（即使换了公司名）  
- 情绪堆砌无机制 / 机制教科书无「我」  
- 四人人群表 + 「仍取决于……」  
- 强情感新模具：每篇「我烦→三点机制→我站队」且 opening/ending 相同
