# 文章结构选择

只选择一种主结构。可以借用另一结构的一个环节，但不要拼成百科全书。所有结构都必须围绕一个核心问题和一句中心判断。

## 目录

1. 事件解读型
2. 科技解释型
3. 企业竞争型
4. 产业博弈型
5. 科技民生型
6. 非投资财经型
7. 快速选择规则

## 1. 事件解读型

适合新闻、企业动作、产品发布、政策公开变化和产业事件。

逻辑顺序：

1. 从关键细节或受影响人群切入。
2. 用必要篇幅交代发生了什么，并注明时间和来源。
3. 解释为什么此时发生。
4. 拆解技术、商业和产业因素。
5. 分析谁获益、谁承担成本。
6. 说明对企业、行业和普通人的实际影响。
7. 给出有条件、有限度的趋势判断。

避免把新闻重述占据全文，也不要用一次发布会推导行业终局。

## 2. 科技解释型

适合 AI、芯片、机器人、自动驾驶、算力、储能等概念。

逻辑顺序：

1. 从普通人的现实场景或工作流程切入。
2. 说明技术解决的具体问题。
3. 用通俗但准确的语言解释原理。
4. 区分演示能力、产品能力和部署能力。
5. 分析成本、可靠性、维护和责任限制。
6. 说明谁会首先采用，谁要承担适配成本。
7. 讨论对工作和生活的实际影响。

不要把术语解释写成教科书，也不要用比喻替代关键准确性。

## 3. 企业竞争型

适合企业、平台、产品和商业模式竞争。

逻辑顺序：

1. 界定竞争发生在哪个市场和客户场景。
2. 交代主要参与者的公开策略。
3. 比较技术、产品、成本、渠道、组织和生态。
4. 分析优势、短板和约束。
5. 解释竞争如何影响员工、客户、供应商与开发者。
6. 说明结果仍取决于哪些变量。
7. 给出阶段性判断，不宣布永久赢家。

不得导向股票买卖、目标价或“概念股”名单。

## 4. 产业博弈型

适合产业链、供应链、标准、专利、人才和跨国企业竞争。

围绕核心技术、研发投入、制造能力、成本、供应链、专利、标准、人才、市场准入、产品生态、商业化和下游适配成本展开。分析公开政策时只说明其产业机制和可验证影响。

不要使用“战争”“决战”“敌我”“封锁必胜”等情绪化叙事，不评价政治制度优劣，不推演军事或未公开国家行动。

## 5. 科技民生型

适合就业、教育、医疗、交通、消费和公共服务。

逻辑顺序：

1. 从普通人的具体体验或工作环节切入。
2. 解释变化背后的技术或产业原因。
3. 识别最先受影响和最容易被忽略的人。
4. 分析便利、成本、风险和责任如何分配。
5. 判断技术改善是否真正到达普通人。
6. 说明新问题和能力边界。
7. 给出克制的现实判断。

不要制造苦难、恐慌或替群体发言。

## 6. 非投资财经型

适合企业收入与成本、供需、产能、价格机制、商业模式、产业周期、技术投入、生产效率、贸易、供应链和经营压力。

逻辑顺序：

1. 定义企业或行业正在面对的经营问题。
2. 拆分收入、成本、供需、产能或效率机制。
3. 解释技术、产品和竞争策略。
4. 分析客户、员工、供应商和下游企业受到的影响。
5. 写清经营优势、限制和风险变量。
6. 给出产业或经营层面的阶段性判断。

不得讨论证券是否值得买、估值吸引力、短期涨跌、买卖点、仓位、收益或投资机会。

## 快速选择规则

- “刚刚发生了什么、为什么现在发生” → 事件解读型。
- “这项技术到底怎么工作、离落地多远” → 科技解释型。
- “两家或多家企业为何采取不同路线” → 企业竞争型。
- “供应链、标准、专利、人才如何相互制约” → 产业博弈型。
- “普通人的工作或服务体验如何变化” → 科技民生型。
- “企业为何赚钱、亏损、降价、扩产或承压” → 非投资财经型。

当多个类型都适用时，以用户真正想回答的问题为准。仍无法判断时，优先选择受影响人群最清晰、证据最充分的类型，而不是追问用户选择结构。
