# 流水线契约简化迁移

用于删除或放宽微信公众号流水线门禁时。目标是让需求、任务状态、运行器、发布边界、cron 提示和测试保持同一契约，避免只改一层后生产任务继续执行旧流程。

## 实施顺序

1. 先写可执行契约：保留项、删除项、允许降级项、幂等边界和验收场景。用户尚在讨论、记录需求或没有明确说开始实施时，只更新需求文档，不改代码、配置、cron 或生产状态。
2. 读取脏工作树，区分现有用户改动；只修改本次契约涉及的文件。
3. 先写独立契约测试并确认旧实现失败（RED）。测试至少覆盖新任务骨架、旧任务迁移、失败降级、幂等和结果不确定状态。
4. 修改任务模型和 CLI：阶段、产物、参数、配置版本与旧 `job.json` 迁移必须一起更新。
5. 修改运行器：删除门禁时同时删除调用、checkpoint、恢复响应、状态摘要和残留字段，不能只跳过主路径。
6. 修改发布边界：业务门禁可降级，但路径越界、账号配置、真实媒体解码、微信 API 错误和 `uncertain` 防重继续保留。
7. 同步 cron：更新任务 prompt、attached skills、`workdir` 和完成检查。多个生产任务共享 `workdir` 时依赖 Hermes 队列串行；不要为了串行把多个长任务塞进一个受单次时限约束的 cron run。只更新定义，不以手动运行生产 cron 作为普通验证。
8. 更新 SKILL.md 与 references，删除旧命令、旧产物和旧恢复规则。主 SKILL 只承载当前契约；历史排障细节放 references，并确保主 SKILL 明确指向仍适用的参考文件。
9. 跑聚焦测试、完整测试和离线同日多运行验收；生产 API 不作为普通回归测试。

## 过程控制

- 先把用户决策固化为需求和验收矩阵，再开始跨文件修改；不要边讨论边扩展门禁。
- 用户明确要求简化时，默认删除误阻塞的业务校验，不把每个边缘条件转成新的硬门禁。安全边界和不可逆副作用单独保留。
- 跨文件迁移过程中不要在未验证状态下宣称完成。被工具轮次中断时，精确报告最后通过的测试与剩余断点，下一轮从断点继续。
- 定时任务、真实微信 API 和生产工作区属于副作用面。除非用户明确要求实跑，只做定义更新、离线桩验收和状态读取。
- 旧测试若专门断言已删除契约，应重写成新契约测试；不能为了让旧测试继续通过而恢复废弃逻辑。保留路径安全、账号隔离、字数、封面、API 错误和 uncertain 等仍有效边界。

## 删除门禁的全局搜索清单

删除一个概念后，搜索并处理：

- 阶段名和 `STAGES`
- `artifacts` 键与默认路径
- CLI 参数、帮助文本和示例命令
- `prepare`、`gate`、`finish`
- 状态摘要和恢复响应
- checkpoint、哈希与复用条件
- completion check 和 cron prompt
- SKILL.md、references 与测试 fixture

兼容旧任务时，`load_job` 可以移除废弃阶段、产物并补新字段；新建任务不得继续写入废弃结构。

## 简化原则

- 选题时效只在选题落库时检查一次，后续生产阶段不要重复检查导致时间漂移阻塞。
- 语义去重由 Agent 读取短 `event_focus` 历史后判断；拿不准时放行。不要用脆弱的字符相似度冒充语义判断。
- 可选正文图失败可以重试后无图继续；缺图引用应清理。路径越界和损坏媒体仍必须阻止上传。
- 图片扩展名或服务器 MIME 与真实字节不一致时，以 magic bytes 加解码结果为准，规范化上传文件名和 MIME；真实损坏才报错。
- 草稿幂等边界是单次 `run_id`：同一成功运行复用结果，新 `run_id` 允许同账号当天再建草稿。
- `draft/add` 结果不确定时保持 `uncertain`，禁止自动重发。
- 删除机械 validate、预览或哈希后，仍保留发布必需的轻量检查：HTML 可读、无占位符、封面可用、账号正确、结果字段完整。

## 验收门槛

没有同时完成以下项目时，只能报告“部分实施”，不得宣称改造完成：

- 新契约测试通过
- 受影响的既有测试已更新并通过
- 完整离线测试通过
- 静态搜索不再出现已删除契约的生产引用
- 两个新 `run_id` 可在同账号同日分别走到草稿调用边界
- 同一成功 `run_id` 二次执行不重复调用草稿 API
- `uncertain` 运行仍被阻止自动重发
- cron prompt 与 SKILL.md 使用新命令和新产物

若受工具轮次或外部阻塞中断，明确列出已改文件、未完成项和最后一次真实测试结果；不要用语法检查替代测试通过。
